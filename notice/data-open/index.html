수시공시

참고
정시 1차 공시의 입력 마감은 지역별 세부 안내가 다를 수 있습니다. 강원 지역 등은 관할 교육청의 별도 공문에서 입력 마감일을 다시 확인해 두는 편이 안전합니다.

3. 처리 흐름과 시스템 구분

·실무 흐름을 한 장으로 정리한 구역

  1. 작성자·확인자 지정: 총괄업무담당자가 공시 항목별 작성자와 확인자를 지정합니다.
  2. 자료 입력 및 작성자 마감: 작성자가 지정 항목을 입력하고 작성자 마감 처리합니다.
  3. 공시자료 확인 및 확인자 마감: 확인자가 입력 내용을 검토하고 확인자 마감 처리합니다.
  4. 관리자 마감 및 상신: 총괄업무담당자가 최종 마감 후 상신합니다.
  5. 공시 후 정정: 공개 후 오류나 개인정보 포함 등이 있으면 정정대장으로 수정합니다.

입력 경로

학교정보공시 → 공시자료관리 → 공시자료등록 → 자료등록

※ 입력 후 작성자 마감 → 확인자 마감 → 총괄 상신 순으로 진행

정정 경로

학교정보공시 → 공시자료관리 → 공시자료정정 → 정정대장관리

※ 공개 후 오류, 개인정보 포함, 수정 필요 시 사용

제외처리와 수시공시를 섞지 않기
공시 대상이지만 입력할 내용이 없는 항목은 제외처리할 수 있지만, 수시 공시항목은 사안이 없으면 등록하지 않는 구조로 보는 편이 맞습니다.

4. 자주 놓치는 부분

헷갈리기 쉬운 포인트

  • 수시공시는 사안이 없으면 등록하지 않으므로, 제외처리 대상과 동일하게 보면 꼬입니다.
  • 공시 후 오류 발견 시 단순 재입력이 아니라 정정대장 처리 흐름을 먼저 확인해야 합니다.
  • 학교회계 예산·결산 공개 일정과 정보공시 일정은 별도 관리가 안전합니다.

제출 전 체크리스트

  • 작성자·확인자·총괄업무담당자 지정 완료
  • 입력 근거자료 보관 여부 확인
  • 검증·수정 기간을 내부 일정표에 반영
  • 수시공시 발생 사안 누락 여부 점검
  • 공개 후 정정 필요 여부 최종 확인

주의 문구

공개일만 보고 준비하면 거의 반드시 늦습니다. 실무에서는 입력 마감일과 검증·수정 기간을 먼저 잡고, 교무·행정·담당부서 자료를 그보다 앞서 회수하는 쪽이 훨씬 안전합니다.

정보공시와 별도로 같이 챙길 공개

예산서와 결산서는 학교운영위원회 심의 후 확정되면 학교 홈페이지 등에 공개하는 별도 공개 일정이 있으므로, 정보공시 일정표와 한 장표에서 완전히 섞어 관리하지 않는 편이 실수 방지에 유리합니다.
